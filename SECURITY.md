# Security

Vilse is currently a private prototype. This public mirror contains only
controlled documentation and does not expose source code, deployment
configuration, datasets, credentials, prompts, or production infrastructure.

## Reporting

If you are given access to a private build, private service, repository, or test
environment and discover a security issue, report it privately to the project
owner. Do not publish exploit details, private URLs, logs, screenshots containing
private data, or reproduction steps in public channels.

## Scope

In scope:

- issues in public documentation that accidentally disclose sensitive
  implementation details
- issues in any private environment you have been explicitly authorized to test

Out of scope:

- scanning, probing, or attacking systems you were not explicitly authorized to
  test
- attempts to reconstruct private code or deployment details from public docs
- social engineering or credential attacks

## Public Mirror Safety

If this mirror appears to contain source code, secrets, private infrastructure
details, datasets, prompts, or other implementation material, treat that as a
release mistake and report it privately.
